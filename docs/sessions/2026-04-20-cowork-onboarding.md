## Решения и договорённости (workflow)

- **Один чат = одна задача / фича / баг.** Длинные «сборные» чаты ухудшают фокус и осложняют ревью.
- **Персистентная память проекта живёт в файлах**, не в истории чата. Между сессиями сохраняются: код, `git`, `CLAUDE.md` (автоинжектится в системный контекст), всё в `docs/`. История диалога — нет.
- **`CLAUDE.md` обновляется**, когда меняется архитектурное решение или закрывается крупный кусок плана.
- **`docs/`** — «второй мозг» проекта. Подразделы:
  - `docs/launch/` (уже есть) — runbooks для лонча.
  - `docs/sessions/` (создан этой сессией) — короткие саммари сессий.
  - `docs/decisions/` (предложен, ещё не создан) — ADR на каждое архитектурное решение.
- **Git**: задачи делаем в feature-ветках, коммиты атомарные, `commit`/`push`/`reset` — только по явной команде пользователя.
